#  Rust 第一节课



Rust 是这几年比较火的编程语言。2006年时，Mozilla的一个员工开始做一个个人项目，后来得到了公司的支持，于2010年发布这个项目。这个项目就叫Rust。



接下来，运行起来Rust的第一个程序吧。打开官网，来看看如何把程序跑起来。



出现了Cargo这个东西，Rust的构建工具和包管理工具。Rust程序就像一篇英文文章。构建工具大致是把这篇英文文章翻译成更底层的表达。



计算机的硬件是一系列的电子电路。在最底层，是用「0」和「1」来表达信息。想象一系列的灯泡开关。如果只有一个灯泡，「0」表示熄灭，「1」表示点亮。那么一个灯泡可以表达两种状态。两个灯泡，则4种状态。十个灯泡，则是2的10次方种状态，即是1024种状态。那么这1024种状态，便可以用来表达数字0到1023。那字符如何表达呢。显然有个方法，把第一个灯泡来区分数字和字符状态。其余9个灯泡，来表达数字值或字符值。当第一个灯泡点亮时，其余九个灯泡表达字符。当其余9个灯泡，都是熄灭的时候，表示字符「a」。



当然，上述说法和现有的计算机表达是有出入的。但当我们去设计自己的计算机的时候，我们可以这么做。如设计一个自己的简易计算器，可以计算1000以内的计算器。我们是可以灵活定义自己的电子电路，定义这些电子电路的信号含义的。



回到构建工具上来。一个程序很复杂，有计算符号、变量和程序固有词汇等等。直接翻译成很多个「0」和「1」给计算机运行还是很困难。所以中间有很多层的表示。Cargo是将Rust程序变成机器码吗，还是别的。
